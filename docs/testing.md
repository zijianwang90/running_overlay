# Testing and Public Fixtures

## Test Layers

- Unit and model tests use Swift Testing under
  `Tests/RunningOverlayTests/`.
- Renderer smoke tests verify that supported asset formats produce pixels.
- Visual regression tests compare selected deterministic renderer output with
  reviewed PNG snapshots.
- Optional real-world FIT tests may be run locally through an environment
  variable, but never gate CI.

## Commands

```sh
./scripts/check.sh
./scripts/test.sh FitFileParserTests
./scripts/visual-test.sh
```

## Fixture Policy

Committed fixtures must be:

- synthetic, authored for this repository, or explicitly licensed;
- free of athlete names, device identifiers, GPS history, and private paths;
- small enough for normal Git storage;
- documented in `Tests/RunningOverlayTests/Fixtures/README.md`.

The repository FIT fixture is generated by
`scripts/generate-fixtures.py`. It has deterministic timestamps and metrics
and contains no GPS coordinates.

External FIT files can be tested without committing them:

```sh
RUNNING_OVERLAY_FIT_SAMPLE=/path/to/local.fit ./scripts/test.sh FitFileParserTests
```

Tests must skip this optional input when the variable is absent.

## Visual Snapshots

Run:

```sh
./scripts/visual-test.sh
```

Regenerate:

```sh
UPDATE_VISUAL_SNAPSHOTS=1 ./scripts/visual-test.sh
```

Snapshot updates are not routine golden-file maintenance. The pull request
must include before/after images and explain why the visual change is correct.
A human reviewer must inspect changed snapshots.

Snapshots should favor deterministic Core Graphics geometry and synthetic
data. Avoid network-backed maps, current time, locale-sensitive content, or
fonts that differ across supported systems.

## Performance Tests

Performance comparisons must use a documented project snapshot with no private
media paths or activity data. Record toolchain, hardware class, settings, and
before/after measurements. Performance snapshots are not committed when they
contain local source paths.
