## Writing Rules

- Add entries to the current `YYYY-MM.md` file in reverse chronological order.
- Use `## YYYY-MM-DD` for each date and `###` for individual changes.
- Record important features, bug fixes, architecture changes, repository-wide
  tooling changes, and the verification performed.
- Create the next monthly file when the calendar month changes.
- If one month exceeds roughly 1,500 lines, split it into chronological parts
  and list them here.
- Historical entries are context, not current design authority. Requirements,
  architecture, development guides, and ADRs take precedence.
