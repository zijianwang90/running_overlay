- Verification: `./scripts/test.sh ExportPerformance`; `./scripts/check.sh`.

### Elevation Chart Export Static-Fill Cache

- Sped up MOV export when several full-profile Elevation Charts are present by
  baking each chart's static geometry once and re-rendering only the cheap
  current-position marker per unique Layer Data sample, instead of rasterizing
  the whole chart (gradient fills, glow blur) every frame.
- Added `ElevationChartLayerVisibility` and an `ElevationChartFillRenderMode` so
  `OverlaySharedElevationChartView` / `ElevationChartOverlayView` can render
  individual layers (chrome+grid+axis+base fill, line+glow, dual-area lower
  fill, marker+axis labels). Defaults reproduce the full preview appearance, so
  preview and normal export are unchanged.
- Added `SwiftUIElevationChartLayerView`, `ExportElevationChartStaticFillCache`,
  `canUseElevationChartStaticFillCache`, the prerender pass, and the per-frame
  composite in `SwiftUIOverlayVideoExporter`. Dark Terrain dual-area charts are
  supported by cropping a baked lower-fill layer to the right of
  `chartAreaWidth * progress` (`elevationChartCutXInImage`) each frame.
- Eligibility excludes progress-to-current mode, big numbers, a visible stats
  bar, and the shared foreground glow, since those change per frame or apply to
  the whole composite and cannot be reproduced by the layered cache.
- Verified pixel parity with `elevationChartStaticFillParity` (mean per-byte
  diff < 0.6, max < 96 across progress 0…1). Local micro-measurement at 1280x720
  showed per-frame render work dropping ~1.86 ms → ~0.79 ms per chart (~2.4x),
  one-time cache build ~37 ms. A full real-media `--benchmark-export` run on a
  multi-chart project is still recommended before release.
- Verification: `./scripts/test.sh elevationChart`; `./scripts/check.sh`.
